Curso Avançado - Introdução ao JavaScript para Adultos

Este curso oferece uma visão completa do JavaScript para adultos em transição de carreira, com explicações detalhadas e exemplos práticos.

Módulo 1: Fundamentos do JavaScript

1. História e uso do JavaScript

JavaScript foi criado em 1995 por Brendan Eich enquanto trabalhava na Netscape. Originalmente chamado de Mocha, depois LiveScript, finalmente foi renomeado para JavaScript como estratégia de marketing. Hoje é a linguagem de programação mais popular do mundo, executando em navegadores, servidores (Node.js), dispositivos IoT e mais.

Curiosidade: JavaScript não tem relação com Java, apesar do nome. A semelhança no nome foi uma decisão de marketing.

2. Sintaxe básica

A sintaxe do JavaScript inclui:

  • Instruções terminam com ponto e vírgula (opcional, mas recomendado)
  • Blocos de código delimitados por chaves {}
  • Comentários de linha única com // e múltiplas linhas com /* */
  • Case sensitive (diferencia maiúsculas de minúsculas)
// Comentário de uma linha
/* Comentário de 
múltiplas linhas */

{
    // Bloco de código
    console.log("Olá, mundo!");
}

3. Variáveis (let, const, var)

JavaScript oferece três maneiras de declarar variáveis:

  • let: variável com escopo de bloco, mutável
  • const: variável com escopo de bloco, imutável (valor não pode ser reatribuído)
  • var: variável com escopo de função, obsoleto (evitar usar)
let nome = "João";  // Pode ser reatribuída
nome = "Maria";

const PI = 3.14159; // Não pode ser reatribuída
// PI = 3.14; // Isso causaria um erro

var idade = 30; // Evitar usar var

Boa prática: Prefira const por padrão e use let apenas quando precisar reatribuir valores. Evite var.

4. Tipos de dados primitivos

JavaScript tem 7 tipos primitivos:

  • string - texto (ex: "Olá")
  • number - números (ex: 42, 3.14)
  • boolean - verdadeiro/falso (true/false)
  • null - valor nulo intencional
  • undefined - valor não definido
  • symbol - valor único e imutável (ES6)
  • bigint - números inteiros muito grandes (ES2020)
let texto = "JavaScript"; // string
let numero = 42;         // number
let ativo = true;        // boolean
let vazio = null;        // null
let naoDefinido;         // undefined
let id = Symbol("id");   // symbol
let grande = 1234567890123456789012345678901234567890n; // bigint

Módulo 2: Operadores e Expressões

1. Operadores aritméticos

JavaScript inclui os operadores aritméticos básicos:

  • + Adição
  • - Subtração
  • * Multiplicação
  • / Divisão
  • % Módulo (resto da divisão)
  • ** Exponenciação (ES2016)
  • ++ Incremento
  • -- Decremento
let soma = 10 + 5;       // 15
let subtracao = 10 - 5;  // 5
let multiplicacao = 10 * 5; // 50
let divisao = 10 / 5;    // 2
let resto = 10 % 3;      // 1
let potencia = 2 ** 3;   // 8

let contador = 0;
contador++; // Incrementa para 1
contador--; // Decrementa para 0

2. Operadores lógicos e comparativos

Operadores comparativos:

  • == Igual (valor)
  • === Igual (valor e tipo)
  • != Diferente (valor)
  • !== Diferente (valor e tipo)
  • > Maior que
  • < Menor que
  • >= Maior ou igual
  • <= Menor ou igual

Operadores lógicos:

  • && AND (e)
  • || OR (ou)
  • ! NOT (não)
// Comparação
console.log(5 == '5');   // true (conversão de tipo)
console.log(5 === '5');  // false (tipos diferentes)
console.log(5 !== '5');  // true

// Lógicos
let idade = 25;
let temCarteira = true;
console.log(idade >= 18 && temCarteira); // true
console.log(idade < 18 || !temCarteira); // false

Boa prática: Sempre use === e !== em vez de == e != para evitar conversões de tipo inesperadas.

3. Precedência de operadores

A precedência determina a ordem em que os operadores são avaliados. Segue a tabela de precedência (do maior para o menor):

  1. Agrupamento: ()
  2. Incremento/decremento: ++ --
  3. Exponenciação: **
  4. Multiplicação/divisão/módulo: * / %
  5. Adição/subtração: + -
  6. Comparação: < <= > >=
  7. Igualdade: == != === !==
  8. AND lógico: &&
  9. OR lógico: ||
  10. Atribuição: = += -= etc.
let resultado = 5 + 3 * 2; // 11 (não 16)
let outroResultado = (5 + 3) * 2; // 16

let a = 1, b = 2, c = 3;
let complexo = a + b * c / a - b; // 1 + 6 / 1 - 2 = 5

Módulo 3: Estruturas Condicionais

1. if, else if, else

As estruturas condicionais permitem executar diferentes blocos de código baseado em condições.

let hora = new Date().getHours();

if (hora < 12) {
    console.log("Bom dia!");
} else if (hora < 18) {
    console.log("Boa tarde!");
} else {
    console.log("Boa noite!");
}

Operador ternário (if/else em uma linha):

let idade = 20;
let status = (idade >= 18) ? "Adulto" : "Menor";
console.log(status); // "Adulto"

2. switch/case

O switch é útil para múltiplas condições baseadas no mesmo valor.

let diaDaSemana = new Date().getDay();
let nomeDia;

switch (diaDaSemana) {
    case 0:
        nomeDia = "Domingo";
        break;
    case 1:
        nomeDia = "Segunda";
        break;
    case 2:
        nomeDia = "Terça";
        break;
    case 3:
        nomeDia = "Quarta";
        break;
    case 4:
        nomeDia = "Quinta";
        break;
    case 5:
        nomeDia = "Sexta";
        break;
    case 6:
        nomeDia = "Sábado";
        break;
    default:
        nomeDia = "Dia inválido";
}

console.log(`Hoje é ${nomeDia}`);

Atenção: Sempre inclua break ao final de cada case, a menos que deseje propositalmente que o código continue executando o próximo case (fall-through).

Módulo 4: Laços de Repetição

1. for, while, do-while

Laços permitem executar um bloco de código repetidamente.

for: Ideal quando se sabe o número de iterações antecipadamente.

// Imprime números de 1 a 5
for (let i = 1; i <= 5; i++) {
    console.log(i);
}

// Iterando sobre um array
let frutas = ['maçã', 'banana', 'laranja'];
for (let i = 0; i < frutas.length; i++) {
    console.log(frutas[i]);
}

while: Executa enquanto uma condição for verdadeira.

let contador = 0;
while (contador < 5) {
    console.log(contador);
    contador++;
}

do-while: Semelhante ao while, mas garante pelo menos uma execução.

let input;
do {
    input = prompt("Digite 'sair' para terminar:");
} while (input !== 'sair');

2. Controle de loops com break e continue

break interrompe completamente o loop, continue pula para a próxima iteração.

// Exemplo com break
for (let i = 0; i < 10; i++) {
    if (i === 5) break;
    console.log(i); // Imprime 0,1,2,3,4
}

// Exemplo com continue
for (let i = 0; i < 5; i++) {
    if (i === 2) continue;
    console.log(i); // Imprime 0,1,3,4
}

Dica: Para loops sobre arrays, considere usar métodos como forEach(), map(), filter() que veremos no módulo 6.

Módulo 5: Funções

1. Declaração e expressão de funções

Existem duas formas principais de definir funções:

Declaração de função: São içadas (hoisted) e podem ser chamadas antes da declaração.

function soma(a, b) {
    return a + b;
}

console.log(soma(2, 3)); // 5

Expressão de função: Atribuída a uma variável, não são içadas.

const multiplica = function(x, y) {
    return x * y;
};

console.log(multiplica(2, 3)); // 6

2. Arrow functions

Introduzidas no ES6, têm sintaxe mais curta e não possuem seu próprio this.

// Forma tradicional
const quadrado = function(x) {
    return x * x;
};

// Arrow function equivalente
const quadrado = x => x * x;

// Com múltiplos parâmetros
const soma = (a, b) => a + b;

// Com múltiplas linhas
const fatorial = n => {
    let resultado = 1;
    for (let i = 2; i <= n; i++) {
        resultado *= i;
    }
    return resultado;
};

3. Parâmetros e retornos

Funções podem ter parâmetros padrão e retornar qualquer tipo de valor.

// Parâmetros padrão
function saudacao(nome = 'Visitante') {
    return `Olá, ${nome}!`;
}

console.log(saudacao()); // "Olá, Visitante!"
console.log(saudacao("João")); // "Olá, João!"

// Retornando múltiplos valores (via array ou objeto)
function operacoes(a, b) {
    return {
        soma: a + b,
        subtracao: a - b,
        multiplicacao: a * b,
        divisao: a / b
    };
}

let resultados = operacoes(10, 5);
console.log(resultados.soma); // 15

Boa prática: Funções devem ter uma única responsabilidade e nomes descritivos que indiquem claramente o que fazem.

Módulo 6: Arrays e Objetos

1. Manipulação de arrays

Arrays são listas ordenadas que podem conter diferentes tipos de dados.

// Criando arrays
let numeros = [1, 2, 3];
let misto = ['texto', 42, true, null];

// Acessando elementos
console.log(numeros[0]); // 1 (índices começam em 0)

// Modificando arrays
numeros.push(4); // Adiciona ao final
numeros.unshift(0); // Adiciona no início
let ultimo = numeros.pop(); // Remove e retorna o último
let primeiro = numeros.shift(); // Remove e retorna o primeiro

console.log(numeros); // [1, 2, 3]

2. Métodos como push, pop, map, filter

Arrays possuem muitos métodos úteis:

let numeros = [1, 2, 3, 4, 5];

// map: transforma cada elemento
let dobrados = numeros.map(n => n * 2);
console.log(dobrados); // [2, 4, 6, 8, 10]

// filter: filtra elementos
let pares = numeros.filter(n => n % 2 === 0);
console.log(pares); // [2, 4]

// reduce: reduz a um único valor
let soma = numeros.reduce((total, n) => total + n, 0);
console.log(soma); // 15

// find: encontra o primeiro que satisfaz a condição
let maiorQue3 = numeros.find(n => n > 3);
console.log(maiorQue3); // 4

// some e every: testam condições
let temPar = numeros.some(n => n % 2 === 0); // true
let todosPares = numeros.every(n => n % 2 === 0); // false

3. Criação e acesso a objetos

Objetos armazenam dados como pares chave-valor.

// Criando objetos
let pessoa = {
    nome: 'Maria',
    idade: 30,
    profissao: 'Engenheira',
    hobbies: ['leitura', 'ciclismo'],
    endereco: {
        rua: 'Rua Principal',
        numero: 123
    },
    saudacao: function() {
        return `Olá, meu nome é ${this.nome}`;
    }
};

// Acessando propriedades
console.log(pessoa.nome); // "Maria"
console.log(pessoa['idade']); // 30 (notação de colchetes)
console.log(pessoa.saudacao()); // "Olá, meu nome é Maria"

// Modificando objetos
pessoa.idade = 31;
pessoa.casado = false; // Adicionando nova propriedade
delete pessoa.hobbies; // Removendo propriedade

// Desestruturação (ES6)
let { nome, idade } = pessoa;
console.log(nome, idade); // "Maria" 31

Dica: Use Object.keys(), Object.values() e Object.entries() para trabalhar com as propriedades de um objeto.

Módulo 7: Manipulação do DOM

1. Seleção de elementos

O DOM (Document Object Model) representa a página HTML como uma árvore de objetos.

// Selecionando elementos
let titulo = document.getElementById('meuTitulo');
let paragrafos = document.getElementsByClassName('paragrafo');
let links = document.getElementsByTagName('a');

// Métodos modernos (querySelector)
let elemento = document.querySelector('#idDoElemento'); // Primeiro elemento que corresponde
let todosElementos = document.querySelectorAll('.classe'); // Todos elementos que correspondem

2. Alteração de conteúdo

Podemos alterar conteúdo, estilos e atributos dos elementos.

// Alterando conteúdo
elemento.textContent = 'Novo texto'; // Apenas texto
elemento.innerHTML = '<strong>Texto</strong> formatado'; // HTML

// Modificando estilos
elemento.style.color = 'blue';
elemento.style.backgroundColor = '#f0f0f0';

// Alterando atributos
elemento.setAttribute('data-info', 'valor');
let valor = elemento.getAttribute('data-info');
elemento.removeAttribute('data-info');

// Adicionando/removendo classes
elemento.classList.add('nova-classe');
elemento.classList.remove('classe-antiga');
elemento.classList.toggle('classe-ativa'); // Adiciona se não existe, remove se existe

3. Eventos e listeners

Eventos permitem responder a interações do usuário.

// Adicionando event listeners
let botao = document.querySelector('#meuBotao');

// Forma 1: propriedade do elemento
botao.onclick = function() {
    console.log('Botão clicado!');
};

// Forma 2: addEventListener (preferida)
botao.addEventListener('click', function(event) {
    console.log('Botão clicado!', event);
    event.preventDefault(); // Evita comportamento padrão
});

// Eventos comuns: click, mouseover, mouseout, keydown, keyup, submit, change, etc.

// Event delegation (para elementos dinâmicos)
document.body.addEventListener('click', function(event) {
    if (event.target.matches('.item-lista')) {
        console.log('Item da lista clicado:', event.target);
    }
});

Boa prática: Sempre use addEventListener em vez de atribuir diretamente a propriedades como onclick, pois permite múltiplos listeners e mais controle.

Módulo 8: Introdução a APIs e Fetch

1. Fazendo requisições com fetch

A API Fetch fornece uma interface para fazer requisições HTTP.

// Requisição GET simples
fetch('https://api.exemplo.com/dados')
    .then(response => {
        if (!response.ok) {
            throw new Error('Erro na requisição');
        }
        return response.json();
    })
    .then(data => {
        console.log('Dados recebidos:', data);
        // Processar os dados aqui
    })
    .catch(error => {
        console.error('Erro:', error);
    });

// Requisição POST com headers e body
fetch('https://api.exemplo.com/dados', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer token123'
    },
    body: JSON.stringify({
        nome: 'João',
        idade: 30
    })
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error(error));

2. Tratamento de JSON

JSON (JavaScript Object Notation) é o formato padrão para troca de dados.

// Convertendo objeto para JSON string
let pessoa = { nome: "Maria", idade: 30 };
let jsonString = JSON.stringify(pessoa);
console.log(jsonString); // '{"nome":"Maria","idade":30}'

// Convertendo JSON string para objeto
let jsonObj = JSON.parse('{"nome":"Maria","idade":30}');
console.log(jsonObj.nome); // "Maria"

// Trabalhando com respostas JSON de APIs
fetch('https://api.exemplo.com/dados')
    .then(response => response.json()) // Converte resposta para objeto
    .then(data => {
        // data já é um objeto JavaScript
        console.log(data);
    });

Dica: Sempre trate erros em requisições fetch e verifique se a resposta foi bem-sucedida (response.ok) antes de processar os dados.

Módulo 9: Promises e Assíncronismo

1. Promises

Promises representam operações assíncronas que podem ser concluídas no futuro.

// Criando uma Promise
function esperar(ms) {
    return new Promise((resolve, reject) => {
        if (ms < 0) {
            reject('Tempo não pode ser negativo');
        } else {
            setTimeout(() => resolve(`Esperou ${ms}ms`), ms);
        }
    });
}

// Usando a Promise
esperar(2000)
    .then(mensagem => {
        console.log(mensagem); // "Esperou 2000ms"
        return esperar(1000); // Encadeamento
    })
    .then(mensagem => console.log(mensagem)) // "Esperou 1000ms"
    .catch(erro => console.error(erro)); // Tratamento de erro

// Métodos úteis
Promise.all([esperar(1000), esperar(2000)])
    .then(() => console.log('Todas promises resolvidas'));

Promise.race([esperar(1000), esperar(2000)])
    .then(() => console.log('A primeira promise resolveu'));

2. Async/await

Sintaxe mais limpa para trabalhar com Promises, introduzida no ES2017.

// Função async sempre retorna uma Promise
async function buscarDados() {
    try {
        // await pausa a execução até a Promise resolver
        let resposta = await fetch('https://api.exemplo.com/dados');
        if (!resposta.ok) throw new Error('Erro na requisição');
        
        let dados = await resposta.json();
        console.log(dados);
        
        // Podemos usar await com qualquer Promise
        let mensagem = await esperar(1000);
        console.log(mensagem);
        
        return dados; // Retornado como Promise resolvida
    } catch (erro) {
        console.error('Falha:', erro);
        throw erro; // Rejeita a Promise retornada
    }
}

// Chamando a função async
buscarDados()
    .then(dados => console.log('Processamento adicional:', dados))
    .catch(erro => console.error('Erro capturado:', erro));

// IIFE (Immediately Invoked Function Expression) com async
(async () => {
    try {
        let resultado = await buscarDados();
        console.log('Resultado:', resultado);
    } catch (erro) {
        console.error(erro);
    }
})();

Importante: await só pode ser usado dentro de funções marcadas como async. Não esqueça de sempre tratar erros com try/catch.
